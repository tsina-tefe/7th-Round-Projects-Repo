# 7th Round Students Projects Repository

Welcome to the **7th Round Students Projects Repository**.
This repository contains projects created by students who are learning at **WabiSkills** and are part of **Group 7**.

The goal of this repository is to collect, organize, and showcase the projects developed during the learning program.

---

## 📌 Purpose of This Repository

This repository is used to:

* Store student projects from **Round 7**
* Help students practice **Git and GitHub collaboration**
* Track student progress and improvements
* Share projects with mentors and other learners
* Build a portfolio of student work

---

## 👨‍💻 Who Should Use This Repository

This repository is intended for:

* Students in **WabiSkills Group 7**
* Mentors reviewing student work
* Contributors helping improve projects

---

## 📂 Repository Structure

Each student should create their own project folder inside the repository.

Example:

```
7th-Round-Projects-Repo
│
├── student-name-project
│   ├── project-files
│   └── README.md
│
├── another-student-project
│   ├── project-files
│   └── README.md
```

---

## 📋 How to Submit Your Project

Follow these steps to add your project:

1. **Fork the repository**
2. **Clone it to your computer**

```
[git clone https://github.com/your-username/7th-Round-Projects-Repo.git](https://github.com/wabiskills/7th-Round-Projects-Repo.git)
```

3. **Create a new folder using your name or project name**

Example:

```
john-portfolio-website
```

4. Add your project files inside the folder.

5. Commit your changes

```
git add .
git commit -m "Add my Round 7 project"
```

6. Push to your fork

```
git push origin main
```

7. Create a **Pull Request** to submit your project.

---

## 📝 Project Requirements

Each project should include:

* Source code
* A project **README.md**
* Clear instructions on how to run the project
* Description of what the project does

---

## ⚠️ Rules

* Do not delete other students' work.
* Keep your project inside your own folder.
* Use clear and meaningful commit messages.
* Make sure your code runs properly before submitting.
